# 管理者ガイド（経理・経営者向け）

Ryohi-Tax Saver を導入する会社の管理者（オーナー・admin・経理）向け運用ガイド。

---

## 1. 管理者の役割分担

| ロール | 権限 |
|---|---|
| **owner** | 全権。会社削除以外すべて可能 |
| **admin** | メンバー管理・規程編集・プラン変更 |
| **approver（上長）** | 申請の承認・差戻し |
| **accountant（経理）** | 承認＋CSV出力＋監査ログ閲覧 |
| **member（一般）** | 自分の出張記録と経費 |

---

## 2. 導入手順（初期1週間）

### Day 1: 会社登録
1. iPhoneアプリ or Web管理画面で代表者がサインアップ
2. 「会社を作成」で自社を登録（自動でownerになる）

### Day 2: 旅費規程の設定
1. Web管理画面 → **旅費規程**
2. **役職別の日当・宿泊費・距離区分**を設定
3. 税務リスク警告（役職間3倍超 / 日額1万円超 / per_km 25円超）に従い調整
4. 「新しいバージョンを発行」で規程を有効化

💡 税理士が監修した社内規程がある場合は、その金額をそのままJSON DSLに入れてください。

### Day 3: メンバー招待
1. Web管理画面 → **メンバー** → 招待メールアドレス入力
2. ロールを選択（一般 / 上長 / 経理 / 管理者）
3. 招待メールのリンクから参加してもらう

### Day 4-6: 試験運用
- 社員数名で実際の出張を記録
- 承認フローを試す
- PDF出力・CSV出力を確認
- 問題があれば社内で調整

### Day 7: 本格運用開始
- 全社員にアプリをインストールしてもらう
- 月初の朝礼等で「申請→承認→精算」のルールを説明

---

## 3. 月次運用フロー

### 月末〜翌月上旬

1. **社員**: 未申請の出張・経費をすべて申請
2. **上長**: 承認タブで一括確認・承認
3. **経理**: Web管理画面で精算書PDF生成＋CSV出力
4. **経理**: freee / MF / 弥生 にCSVをインポート
5. **経理**: 支払手続→各社員の「支払済」ステータスに更新

### 四半期・年次

- Web管理画面 → **監査ログ** で全操作履歴を保存
- 監査ログは改ざん不能（ハッシュチェーン）
- 税務調査時に証跡として提出可能

---

## 4. 旅費規程の税務的な注意

顧問税理士からのアドバイス（井上税理士監修）：

### 🟥 やってはいけない
- 代表取締役だけ日当5万円など**役職間で3倍以上の格差**
- 日当1万円超の設定（給与認定リスク）
- 役職によって恣意的な金額（実績や勤続年数で変える等）

### 🟨 注意すべき
- per_km は **15〜25円/km** が実務水準。50円超は警告
- 飛行機・新幹線は距離計算ではなく**実費精算**推奨
- 日当と交通費実費の**二重計上**に注意

### 🟩 推奨設定例

| 役職 | 日当（1万円上限） | 宿泊手当 |
|---|---|---|
| 代表取締役 | 5,000〜7,000円 | 10,000〜15,000円 |
| 取締役 | 4,000〜5,000円 | 8,000〜12,000円 |
| 従業員 | 2,000〜4,000円 | 7,000〜10,000円 |

---

## 5. 電子帳簿保存法への対応

- 本アプリは電帳法の3要件（真実性・可視性・検索性）に対応する設計
- **Proプラン**: 事務処理規程（`docs/legal/jimusho_kitei_template.md`）と併用で要件を満たす運用が可能
- **Enterpriseプラン**: 認定タイムスタンプが自動付与される
- 保存期間: **7年間**（欠損金繰越時は10年）

---

## 6. 会計ソフト連携

- 対応フォーマット: freee / マネーフォワード / 弥生 / 汎用CSV
- **freee APIダイレクト連携**（設定タブから有効化、OAuth2）
- インポート前に、各社の **勘定科目IDマッピング** を1回だけ設定

---

## 7. 管理者のセキュリティチェック

週1回、以下を確認:

- [ ] 不正な申請パターンがないか（深夜・休日の突発申請等）
- [ ] 差戻しが多いユーザー
- [ ] 退職予定者のアカウントを無効化
- [ ] 月次で監査ログを保全（PDFエクスポート）
